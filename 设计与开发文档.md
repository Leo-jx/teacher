## 1. 需求规格说明

### 1.1 项目背景

随着人工智能技术的快速发展，将AI辅助工具集成到软件开发流程中已成为趋势。本项目旨在为程序员提供一个集编程技术问答、代码分析、语法学习、算法讲解、错误解读、题目练习于一体的AI辅助平台，帮助开发者高效解决开发过程中遇到的技术问题。

### 1.2 功能需求

#### 1.2.1 核心功能

| 功能模块 | 功能描述 |
|---------|---------|
| **智能对话** | 支持自然语言编程技术问答，涵盖主流编程语言、框架、数据库等知识 |
| **代码工具** | 提供代码纠错、代码分析、语法学习、算法讲解、错误解读、题目练习六大功能 |
| **用户账户** | 支持用户注册、登录、退出登录功能 |
| **对话历史** | 保存用户对话记录，支持历史对话回顾与删除 |
| **响应式设计** | 支持桌面端与移动端访问 |

#### 1.2.2 代码工具详细需求

##### 1.2.2.1 代码纠错
- 支持多种编程语言（Java、Python、JavaScript、C/C++、SQL、Vue）
- 自动检测代码中的语法错误、逻辑错误、潜在bug
- 提供修复建议与优化方案

##### 1.2.2.2 代码分析
- 分析代码结构与逻辑
- 指出代码复杂度问题
- 提供性能优化建议

##### 1.2.2.3 语法学习
- 按编程语言分类
- 支持关键词查询语法说明
- 提供语法使用示例

##### 1.2.2.4 算法讲解
- 按算法类型分类（排序、搜索、动态规划、图、树等）
- 详细讲解算法原理
- 提供实现代码示例

##### 1.2.2.5 错误解读
- 支持解析错误信息与异常堆栈
- 分析错误原因
- 提供解决方案

##### 1.2.2.6 题目练习（新增）
- **题分类**：基础语法、数据结构、算法思维、数据库、框架应用、综合实战
- **难度级别**：入门、进阶、挑战
- **语言支持**：Java、Python、JavaScript、C/C++、SQL、Vue
- **练习数量**：可选择1题、3题、5题
- **进度跟踪**：记录用户完成练习数量与进度
- **题目内容**：题目描述、输入输出要求、示例、解题思路、参考代码、常见错误提示、优化建议

#### 1.2.3 技术知识导航

提供结构化的技术知识分类，包括：
- MySQL数据库
- Java与框架
- Python编程
- C/C++开发
- 小程序与跨平台
- AI与前端后端

### 1.3 非功能需求

| 需求类型 | 详细要求 |
|---------|---------|
| **性能要求** | 页面加载时间 < 3秒，AI响应时间 < 30秒 |
| **兼容性** | 支持主流浏览器（Chrome、Firefox、Safari、Edge） |
| **安全性** | 用户密码加密存储，JWT token认证 |
| **可用性** | 界面简洁美观，操作流程直观 |
| **可扩展性** | 代码结构清晰，便于功能扩展 |

## 3. 数据库设计说明

### 3.1 数据库概述

本项目使用 SQLite 作为数据库（Cloudflare D1），采用关系型数据模型设计。

### 3.2 数据表设计

#### 3.2.1 用户表 (users)

| 字段名 | 类型 | 约束 | 说明 |
|-------|------|------|------|
| id | INTEGER | PRIMARY KEY, AUTOINCREMENT | 用户ID |
| username | VARCHAR(50) | UNIQUE, NOT NULL | 用户名 |
| email | VARCHAR(100) | UNIQUE, NOT NULL | 邮箱 |
| password_hash | VARCHAR(255) | NOT NULL | 密码哈希 |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| last_login | DATETIME | NULL | 最后登录时间 |

#### 3.2.2 对话表 (conversations)

| 字段名 | 类型 | 约束 | 说明 |
|-------|------|------|------|
| id | INTEGER | PRIMARY KEY, AUTOINCREMENT | 对话ID |
| user_id | INTEGER | NOT NULL, FOREIGN KEY | 用户ID |
| title | VARCHAR(100) | NOT NULL | 对话标题 |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 更新时间 |

#### 3.2.3 消息表 (messages)

| 字段名 | 类型 | 约束 | 说明 |
|-------|------|------|------|
| id | INTEGER | PRIMARY KEY, AUTOINCREMENT | 消息ID |
| conversation_id | INTEGER | NOT NULL, FOREIGN KEY | 对话ID |
| role | VARCHAR(20) | NOT NULL | 角色(user/assistant) |
| content | TEXT | NOT NULL | 消息内容 |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

## 5. 核心功能实现说明

### 5.1 前端架构

#### 5.1.1 DevAssistant 类

主要职责：
- 初始化应用
- 管理状态（用户、对话、消息等）
- 处理用户事件
- 调用后端API

主要方法：
- `init()` - 应用初始化
- `sendMessage()` - 发送消息
- `createNewChat()` - 新建对话
- `loadChatHistory()` - 加载历史对话
- `renderMarkdown()` - Markdown渲染
- `openCodeTool()` - 打开代码工具面板
- `startPractice()` - 开始题目练习

### 5.2 后端API设计

| API端点 | 方法 | 功能描述 |
|--------|------|---------|
| `/api/auth/register` | POST | 用户注册 |
| `/api/auth/login` | POST | 用户登录 |
| `/api/auth/me` | GET | 获取当前用户信息 |
| `/api/conversations` | GET | 获取对话列表 |
| `/api/conversations` | POST | 创建新对话 |
| `/api/conversations/:id` | GET | 获取对话详情 |
| `/api/conversations/:id` | DELETE | 删除对话 |
| `/api/chat` | POST | 发送AI对话请求 |
| `/api/config` | GET | 获取配置信息 |

### 5.3 题目练习功能实现

**核心逻辑**：
1. 用户在练习面板选择参数（分类、难度、语言、数量）
2. 点击"开始练习"触发 `startPractice()`
3. 根据选择构建详细的prompt请求AI生成题目
4. 进度更新到localStorage

## 7. 安全设计

### 7.1 密码安全

- **加密算法**：SHA-256 + 随机盐值
- **存储方式**：不存储明文密码，仅存储 salt:hash
- **实现位置**：[public/_worker.js](file:///c:/Users/17570/Desktop/chat/public/_worker.js#L27-L49)

### 7.2 身份认证

- **认证方式**：JWT (JSON Web Token)
- **Token过期**：7天有效期
- **实现位置**：[public/_worker.js](file:///c:/Users/17570/Desktop/chat/public/_worker.js#L62-L89)
